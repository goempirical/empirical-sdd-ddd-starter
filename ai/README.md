# AI Engineering Principles

This repository follows a lightweight AI-native engineering workflow.

## Core Ideas

- Specs are first-class artifacts
- Context quality matters
- Small incremental delivery wins
- AI works better with structure
- Simplicity beats complexity
- Roles improve clarity
- Documentation is executable guidance

## Engineering Philosophy

Prefer:
- clarity
- maintainability
- simplicity
- composability
- incremental progress

Avoid:
- overengineering
- giant specs
- speculative abstractions
- unnecessary complexity

## Important Mindset

The goal is not replacing engineers with AI.

The goal is helping engineers:
- think more clearly
- move faster
- collaborate better
- create more predictable delivery workflows
